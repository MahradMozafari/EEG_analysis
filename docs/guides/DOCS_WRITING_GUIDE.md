# Documentation Writing Guide

Use this guide when you continue documenting the repository.

## Recommended documentation order

1. README overview
2. Architecture
3. Methods
4. File contracts
5. API pages per module
6. Example runs
7. Troubleshooting

## Template for each module page

- What problem this module solves
- Inputs
- Outputs
- Side effects
- Internal dependencies
- Example usage
- Failure modes
- Testing notes

## Style rules

- Prefer short direct sentences.
- Name the exact artifact being produced.
- Separate assumptions from observed behavior.
- State whether a diagram is static-analysis-based or conceptual.
- When discussing a metric, say why it matters for EEG causal analysis.
